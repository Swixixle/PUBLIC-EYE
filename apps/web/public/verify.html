How verification works

1
We fetch the receipt from the PUBLIC EYE database using your ID. The receipt contains the full analysis payload — every claim, source, and framing note.
2
We canonicalize the payload using RFC 8785 JCS — the same deterministic JSON serialization used at signing time. Key order, whitespace, and encoding are all normalized before hashing.
3
We verify the signature using the PUBLIC EYE Ed25519 public key (visible below). If the payload was altered even by a single character, the signature fails.
4
You can verify offline — download the raw JSON and run openssl pkeyutl against the public key yourself. No trust in PUBLIC EYE required.
